安全边界

Security model

可信内网工具，不是公网 SaaS Trusted-network tool, not public SaaS

OpenKB 1.x 无登录鉴权。适合本机、局域网、私有 Docker — 请勿将 :8788 裸奔到公网。 OpenKB 1.x has no auth. Use on localhost, LAN, or private Docker — do not expose :8788 to the public internet.

✅ OK

  • 本机 127.0.0.1localhost 127.0.0.1
  • 局域网 / VPN / 私有 DockerLAN / VPN / private Docker
  • 反代 + TLS + 自建鉴权Reverse proxy + TLS + your auth

❌ Not v1.x

  • 公网 VPS 裸奔 0.0.0.0:8788Public VPS on 0.0.0.0:8788
  • 多租户 SaaS（v2 工程）Multi-tenant SaaS (v2 scope)
  • 无鉴权的演示站写操作Writable public demo without auth

快速开始

Quick start

Clone → uv → serve Clone → uv → serve 

git clone https://github.com/GoDiao/openkb.git
cd openkb
uv sync --extra dev
cd web && npm ci && npm run build
export OPENKB_ROOT="$(pwd)" OPENKB_AGENT_ID="my-agent"
uv run openkb serve --port 8788
# → http://127.0.0.1:8788
